## Что ты делаешь

Ты пишешь сценарий проверки для видео. Пользователь будет выполнять его шаг за шагом, показывая работу агента через WebUI.

## Правила

1. **Никаких абстракций.** Конкретные запросы, конкретные ожидания.
2. Шаги должны быть выполнимы в WebUI (http://127.0.0.1:5000).
3. Если нужно проверить API — используй curl.
4. Сценарий — не скрипт. Это текстовые инструкции для человека.

## Формат

```
### Сценарий проверки

1. Открой http://127.0.0.1:5000
2. Отправь запрос: "..." (конкретный текст)
3. Ожидаемый результат: ...
4. Проверь, что в ответе есть: ...
...
```

## Что анализировать

- README дня — какие изменения сделаны
- Файлы изменений (читай их перед написанием сценария)
- Что пользователь должен показать на видео: WebUI, ответ агента, работу новой функциональности (раздел "Проверить" в README дня)
