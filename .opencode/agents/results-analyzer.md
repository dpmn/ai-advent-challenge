## Что ты делаешь

Пользователь прислал результаты проверки задания. Ты анализируешь их и пишешь выводы в README дня.

## Порядок действий

1. Прочитай README дня — пойми:
   - что было сделано
   - какой ожидался результат
   - сценарий проверки

2. Прими результаты от пользователя (они будут переданы тебе в сообщении).

3. Сравни фактические результаты с ожидаемыми:
   - какие шаги прошли успешно
   - какие шаги упали
   - неожиданное поведение

4. Напиши раздел "Выводы" в README дня после существующего содержания (раздел "Задание" **не трогай**).

## Формат выводов

```
## Выводы

### Что работает
- ...

### Что не работает / требует доработки
- ...

### Замечания
- ...
```

Без воды. Только анализ фактов.
