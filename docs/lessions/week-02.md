# Неделя 2. Инструментарий агентов

Во втором учебном модуле рассматривается «внутренняя кухня» ИИ-агентов, а именно то, как они управляют контекстом, памятью и инструментами для эффективного делегирования задач пользователем.

## 1. Управление контекстом (Context Management)

У нейросетей нет встроенной памяти: каждая сессия начинается с чистого листа. Иллюзия того, что модель «помнит» пользователя, создается за счет подгрузки данных в каждую сессию. Внутри контекста постоянно происходит калибровка весов (оценка ответов, направление дискуссии).

Основные способы контроля контекста:

- **History Compression:** Сжатие истории диалога в краткий вывод (summary) при переполнении контекстного окна.
- **Sliding Window:** Сдвиг «окна» на последние сообщения, отсекая старые данные.
- **Vector Embeddings:** Выдёргивание только нужных знаний из базы данных в ходе диалога, что экономит место в контекстном окне.
- **Context Layering:** Разделение знаний на слои: стратегический (весь проект), оперативный (текущая фича) и конкретная задача.
- **Topic Branches:** Дробление контекста по темам, как это делают современные промышленные нейронки (например, ChatGPT).

## 2. Управление памятью (Memory Management)

Помимо кратковременной сессионной памяти, агентам необходима **долговременная память**.

- **Vector Based Memory:** Хранение фактов в виде эмбеддингов в векторной БД. Это позволяет сохранять нюансы, но требует дополнительных мощностей для перевода векторов в текст.
- **Rule-based Memory:** Память на основе жестких правил и термов (нечёткая логика). Дает высокую детерминированность, но она негибкая.
- **Self-reflective Memory:** Сохранение семантических смыслов (настроение пользователя, его предпочтения в общении). Это создает ощущение, что нейронка «узнает» человека.
- **Stores summary** - (компрессированная память) - по сути суммаризация контекстного окна. Сжимаем все сообщения, чтобы было понятно, что происходило на последнем этапе, и закидываем в память. Потребляем мало токенов, но может упускать важные детали.
- **Layer-based memory** (слоистая память) - разбиваем контекст на слои, приоритезируем и сохраняем. Масштабируемый и точный подход, но сложен в реализации.

## 3. Планирование и работа с инструментами (Tool Orchestration)

Для создания своего агента (особенно код-ассистента) критически важна стадия **планирования**: 

- **Goal detection** - определение целей
- **Task decomposition** - разбитие на задачи
- **Execution stratrgy** - выбор стратегии исполнения
- **Replanning** - перепланирование, если понятно, что стратегия не ложится на задачи
- **Evaluation** - реализация


Процесс работы с инструментами включает в себя:

- Выбор и вызов инструмента (Selection & Invocation).
- **Интерпретацию результатов:** Понимание того, подходит ли ответ инструмента под текущую цель.
- Обработку ошибок и обновление стейта (состояния) агента.

## 4. Оптимизация и лайфхаки

- **Сужение контекста:** Необходимо четко указывать стек технологий, язык и прикреплять конкретные файлы, чтобы агент не «плавал» в бесконечном объеме знаний.
- **Stage-based execution:** Разбиение работы на стадии и передача каждого этапа новому, «чистому» агенту с полным контекстным окном.
- **Промпты «Pro Max»:** Использование метапромптинга (когда нейросеть генерирует инструкции для другой нейросети) вместо простых текстовых запросов.
- **Экономия токенов (TOON vs JSON):** Стандартный JSON тратит много токенов из-за дублирования имен полей. Использование формата **TOON** позволяет сократить расход токенов на 30–60% и повысить точность ответов

## 5. Комбинирование управления контекстом и памятью

```text
// Basic Assistant

Context

// Smart Assistant

Context
+ Session memory
+ Vector memory
+ Rules

// Advanced Agent

Hierarchical memory
+ Self-reflection
+ Vector retrieval
```

- **Базовый ассистент:** Использует только **контекст**, который накидывается в текущую сессию.
- **Умный ассистент (уровня GPT-4o):** Комбинирует **сессионную память** с **векторной памятью** и жесткими **правилами**, подмешивая их в контекст для получения более точных ответов.
- **Продвинутый помощник (концепция GPT-5):** Использует **иерархическую память** (разбитую на слои), **self-reflection** (сохранение смыслов и настроения) и **векторный поиск** (извлечение конкретных фактов). Такая комбинация позволяет агенту вести себя почти как человек, помня факты из жизни пользователя и подстраиваясь под его стиль

### Слоистая структура

Одной из наиболее эффективных и масштабируемых техник является объединение **слоев контекста** и **слоистой памяти**:

- Вместо того чтобы загружать в модель всю историю, информация бьется на **стратегический слой** (глобальная цель проекта), **оперативный слой** (текущая крупная фича) и **слой конкретной задачи**.
- В зависимости от запроса система приоритизирует эти данные и подкидывает в контекстное окно только те слои, которые необходимы прямо сейчас. Это делает работу агента очень точной, хотя и сложной в реализации
