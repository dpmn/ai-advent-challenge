---

### 🧠 Архитектура бота:

#### 1. **Telegram Bot API**
- Получает голосовые сообщения от пользователя.
- Отправляет их в **ASR-сервис** (например, Whisper).
- После получения транскрипции и анализа — отправляет обратную связь (где возможно — голосом).

#### 2. **ASR-сервис (Whisper или Google Web Speech)**
- Транскрибирует голосовое сообщение в текст.
- Возвращает текст в **бэкенд-сервер**.

#### 3. **Бэкенд-сервер (FastAPI / Flask)**
- Получает текст от ASR.
- Передаёт текст в **LLM** (например, GPT-3.5, Llama3).
- LLM анализирует:
  - Грамматику,
  - Произношение (если возможно),
  - Дает рекомендации.
- Бэкенд формирует ответ (текст или голосом) и отправляет его в Telegram.

#### 4. **LLM (GPT-3.5, Llama3, phi-3 и т.п.)**
- Анализирует текст, даёт обратную связь.
- Может быть:
  - **Локально** (на GPU-инстансе Cloud.ru),
  - **Облачно** (OpenAI API, Azure, etc.).

#### 5. **База данных (PostgreSQL / MongoDB)**
- Хранит:
  - Пользовательские данные (ID, имя, уровень),
  - Историю голосовых сообщений,
  - Результаты анализа,
  - Прогресс (например, уровень A1–C1, статистика).

---

### 📂 Как хранить прогресс пользователя:

#### Пример структуры таблицы в PostgreSQL:

```sql
CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    telegram_id BIGINT UNIQUE NOT NULL,
    username TEXT,
    level VARCHAR(5) DEFAULT 'A1', -- A1, A2, B1, B2, C1
    last_activity TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE user_progress (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(user_id),
    audio_file TEXT, -- путь к файлу или UUID
    transcription TEXT,
    analysis TEXT, -- результат анализа от LLM
    feedback TEXT, -- текстовая или голосовая обратная связь
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

### 🔄 Поток данных:

1. Пользователь отправляет голосовое сообщение в Telegram.
2. Telegram Bot API получает его и передаёт на бэкенд.
3. Бэкенд сохраняет голосовой файл (на Cloud.ru или в БД).
4. ASR транскрибирует аудио в текст.
5. Текст отправляется в LLM для анализа.
6. LLM возвращает оценку и рекомендации.
7. Бэкенд формирует ответ (текст или голосом) и отправляет в Telegram.
8. Результаты сохраняются в БД.
