### 🧩 **1. Архитектура системы**

```
[Пользователь] → [Telegram Bot] → [API-сервер] → [ASR] → [LLM] → [БД]
```

---

### 🧱 **2. Компоненты и их взаимодействие**

#### 1. **Telegram Bot (aiogram)**
- Получает голосовые сообщения от пользователя.
- Передаёт их на сервер для обработки.

#### 2. **API-сервер (FastAPI или Flask)**
- Принимает голосовые сообщения.
- Сохраняет их временно (например, в `tmp/`).
- Вызывает **ASR-сервис** для транскрипции.
- Отправляет транскрибированный текст в **LLM**.
- Получает обратную связь и отправляет её пользователю.

#### 3. **ASR-сервис (Whisper / Google STT)**
- Транскрибирует голосовые сообщения в текст.
- Возвращает текст для анализа.

#### 4. **LLM (GPT, Hugging Face, LLaMA)**
- Анализирует текст:
  - Проверяет грамматику.
  - Оценивает произношение (по тексту).
  - Генерирует обратную связь.
- Возвращает результаты в API-сервер.

#### 5. **База данных (SQLite, Firebase, PostgreSQL)**
- Хранит:
  - Прогресс пользователя (уровень, количество выполненных заданий).
  - Историю диалогов и обратной связи.
  - Настройки (уровень сложности, язык, тайминги).

---

### 📁 **3. Как хранить прогресс пользователя**

#### Пример структуры БД (SQLite):
```sql
CREATE TABLE users (
    user_id INTEGER PRIMARY KEY,
    username TEXT,
    level TEXT,         -- A1, A2, B1, B2, C1
    progress TEXT,      -- JSON или текст с прогрессом
    last_activity DATETIME
);

CREATE TABLE dialog_history (
    id INTEGER PRIMARY KEY,
    user_id INTEGER,
    text TEXT,
    feedback TEXT,
    timestamp DATETIME,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

---

### 📦 **4. Пример потока данных**

1. Пользователь отправляет голосовое сообщение в Telegram.
2. Бот получает его и сохраняет в временной папке.
3. API-сервер запускает ASR (например, Whisper) для транскрипции.
4. Транскрибированный текст отправляется в LLM (GPT) для анализа.
5. LLM возвращает обратную связь.
6. API-сервер отправляет ответ пользователю через Telegram.
7. Прогресс пользователя записывается в БД.
