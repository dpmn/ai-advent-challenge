### 11.1. Зачем тестировать приложение?

Тестирование — важный этап разработки, который позволяет убедиться в корректности работы приложения и минимизировать количество ошибок. В веб-разработке тестирование помогает:

- Проверить корректность обработки запросов и ответов.
- Убедиться в правильном выполнении бизнес-логики.
- Обеспечить стабильность при внесении изменений в код.

### 11.2. Виды тестов

Существует несколько типов тестов:

- **Unit-тесты** — проверяют отдельные функции или методы.
- **Интеграционные тесты** — проверяют взаимодействие нескольких компонентов.
- **Системные тесты** — имитируют поведение реального пользователя.

### 11.3. Инструменты для тестирования Flask

Для тестирования веб-приложений на Flask можно использовать следующие инструменты:

- **pytest** — популярная система для написания тестов.
- **Flask-Testing** — расширение Flask, упрощающее написание тестов.
- **Selenium** — для автоматизации браузерных тестов.

### 11.4. Пример написания теста

Рассмотрим пример теста для простого маршрута, который возвращает приветствие:

python
import pytest
from flask import Flask

def test_hello_route():
    app = Flask(__name__)

    @app.route('/hello')
    def hello():
        return "Hello, World!"

    with app.test_client() as client:
        response = client.get('/hello')
        assert response.status_code == 200
        assert response.data == b"Hello, World!"
