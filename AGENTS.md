# AI Advent Challenge — главный конфиг сессии

## О проекте

Это рабочий журнал курса-челленджа AI Advent Challenge #8, цель которого научиться быстро использовать инструменты ИИ.
Здесь находится код заданий, конспекты лекций и прогресс прохождения.

### Правила челленджа

- Лекции с новой темой выходят в начале каждой недели.
- Задания выходят **пн–пт в 14:00 МСК**. Всего 7 недель.
- Формат сдачи каждого задания: **видео + код**, причём **видео важнее кода**.
- Результаты вношу в общую таблицу курса.
- **Не сдал 5 заданий за неделю → выбываю.**
- Опоздание по сдаче — максимум **15 минут**.
- Принцип курса: всё делаем **через ИИ, код руками не пишем**.

## Правила инициализации новой сессии

- Прочитай [progress.md](docs/progress.md), чтобы понимать какие были задания и на каком я задании сейчас (todo/done).
- Прочитай выжимку по лекции текущей недели задания, чтобы понимать его контекст.
- Изучи README предыдущего дня. Возможно контекст оттуда понадобится в текущем задании.


## Рабочий процесс выполнения задания

- При получении нового задания я буду создавать файл week-xx -> day-xx -> README.md, в котором пропишу суть задания.
- Я попрошу тебя выполнить задание дня X.
- Сделай доработку кода в соответствии с заданием (в основном сейчас это доработка класса агента).
- Добавь py-файл для запуска CLI с агентом по аналогии с предыдущими днями. Возможно понадобиться прописать новые команды.
- Сделай минимальную проверку работоспособности кода (запустив файл CLI из предыдущего шага, не обязательно по сценарию задания), используя базовую дешёвую модель. Если что-то не так, доделывай, пока не получишь рабочий вариант.
- Запиши в README соответствующего дня в каких файлах и какие доработки кода были выполнены (кратко).
- В этом же README запиши для меня сценарий для проверки в соответствии с заданием. Но не запускай его сам.
- Я сам прогоню сценарий проверки и потом пришлю тебе результаты.
- Тебе нужно провести анализ результатов и записать выводы в этом же README. Раздел "Задание" **не редактируй**!
- Актуализируй структуру проекта в [AGENTS.md](AGENTS.md), если понадобится.
- Актуализируй статус по заданию в [progress.md](docs/progress.md).


## Стек и окружение

- Операционная система: Windows 11, opencode работает на WSL.
- Язык программирования: Python 3.12.
- API для кода заданий: **Cloud.ru, сервис Foundation Models** (оплата в рублях, OpenAI-совместимый доступ).
  - Спецификация API: https://cloud.ru/docs/foundation-models/ug/topics/api-ref__specs
  - base_url: `https://foundation-models.api.cloud.ru/v1`
  - Auth: заголовок `Authorization: Bearer <CLOUDRU_SECRET_KEY>`
  - Ключ: в `.env` как `CLOUDRU_SECRET_KEY` (НЕ коммитить, см. `.gitignore`).
- ID моделей, которые можно использовать в запросах
  - Qwen/Qwen3-30B-A3B - базовая модель (дёшево), для простых задач
  - Qwen/Qwen3-Coder-Next - для задач средней сложности
  - MiniMaxAI/MiniMax-M2.5 - для тяжёлых задач
