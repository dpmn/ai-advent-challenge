# Неделя 5 — День 21. Индексация документов

## Задание

**Возьмите набор документов:**
- README / статьи / код / pdf → текст
- минимум 20–30 страниц текста суммарно (или эквивалент в коде)

**Реализуйте пайплайн индексации:**
- разбиение на чанки (chunking)
- генерация эмбеддингов
- сохранение индекса (FAISS / SQLite / JSON)

**Усиление:**
- добавьте метаданные к каждому чанку (source, title/file, section, chunk_id)
- сделайте минимум 2 стратегии chunking и сравните их:
  - по фиксированному размеру
  - по структуре (заголовки/разделы/файлы)

**Результат:**
- Локальный индекс документов с эмбеддингами + метаданные + сравнение 2 стратегий chunking

## Решение

### Созданные файлы

| Файл | Назначение |
|---|---|
| `ragger/__init__.py` | Пакет |
| `ragger/__main__.py` | Точка входа `python -m ragger` |
| `ragger/pipeline.py` | Главный пайплайн (5 шагов) |
| `ragger/document_loader.py` | Сбор .md/.py из проекта, извлечение текста/docstring |
| `ragger/chunking.py` | Две стратегии: fixed-size (1000 токенов, overlap 100) и structural (по заголовкам) |
| `ragger/embedder.py` | Cloud.ru `/v1/embeddings`, модель `openai/text-embedding-3-small` |
| `ragger/indexer.py` | FAISS IndexFlatIP + metadata.json |
| `ragger/compare.py` | Сравнение стратегий |
| `ragger/data/fixed/` | Индекс фиксированного чанкинга |
| `ragger/data/structural/` | Индекс структурного чанкинга |

### Как запустить

```bash
python ragger/pipeline.py
```

### Источники документов (36 шт., ~136K символов)

- `docs/lessions/week-01.md` … `week-05.md` — конспекты лекций
- `week-XX/day-YY/README.md` — все README заданий (дни 1–21)
- `AGENTS.md` — конфиг сессии
- `agents/*.py` — docstrings модулей агента

### Результаты сравнения

```
======================================================
Метрика                       Fixed        Structural
======================================================
Количество чанков              59           36
Средняя длина (токенов)        615          945
Мин длина (токенов)            9            9
Макс длина (токенов)           1000         1994
======================================================
```

- **Fixed-size:** более равномерные чанки (~615 токенов), предсказуемый размер
- **Structural:** крупнее (~945 токенов), сохраняет целостность разделов, но возможны перекосы (от 9 до 1994)

### Зависимости (добавлены в requirements.txt)

- `numpy>=1.26.0`
- `faiss-cpu>=1.8.0`
