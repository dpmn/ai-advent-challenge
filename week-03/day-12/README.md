# Неделя 3 — День 12. Персонализация ассистента

## Задание

**Добавить персонализацию поверх модели памяти:**
- создай профиль пользователя
- опиши предпочтения (стиль, формат, ограничения)
- подключи профиль к каждому запросу

**Примечание:** Персонализация была полностью реализована в день 11 (класс `Profile`, команды `/profile`, `/profile set`, `/profile new`, `/profiles`, инжекция в system prompt). Доработка кода не требуется.

## Выполнение

Изменённые файлы — нет. Персонализация уже реализована в день 11 (см. `agents/jarvis.py`, класс `Profile` и метод `chat()`).

## Сценарий для видео

Запись через WebUI (http://127.0.0.1:5000). Демонстрируем, как **разные профили** меняют стиль общения ассистента.

### Подготовка к записи

1. Убедись, что `.env` содержит `CLOUDRU_SECRET_KEY`.
2. Запусти Flask: `python webui/app.py`
3. Открой в браузере: http://127.0.0.1:5000
4. Удали старые сессии через ✕.
5. Сбрось профиль на default: `/profile new default` (чтобы начать с чистого листа).
   **Начать запись видео.**

## Результаты проверки

### Профиль 1 — Гопник ✅
- **style:** Пацанский сленг — «слыши, братан», «реально так», «не шмогла», «тема», «по жизни» — отработан полностью.
- **format:** Коротко, цитата «Слыши, братан...» + объяснение + совет «Тема, братан, по жизни» — соблюдён.
- **constraints:** Без мата, лаконично — соблюдено.

### Профиль 2 — Пират ✅
- **style:** «Йо-хо-хо», «тысяча чертей», «чёрт побери», «полный вперёд», морские метафоры (карты океана, корабль, пираты, волки) — отлично.
- **format:** Приветствие с «Йо-хо-хо», каждый абзац как история, совет «Полный вперёд, друг!» — соблюдён.
- **constraints:** Слова «инновации» и «цифровизация» не использованы. ✅

### Профиль 3 — Романтик-поэт ✅
- **style:** Возвышенный слог, «чудеса», «тени», метафоры («как дети», «как тени») — есть. Обращение «милый друг» не появилось, но общий тон выдержан.
- **format:** Четверостишие + прозаическое пояснение в скобках — соблюдён.
- **constraints:** Без сленга, без смайликов, без банальностей — соблюдено.

## Ожидаемые критерии

1. **Разные ответы для разных профилей** — три сессии с разными профилями отвечают на один вопрос в трёх разных стилях (гопник, пират, поэт)
2. **Автоматический учёт профиля** — ассистент не спрашивает «в каком стиле писать?», а применяет поля style/format/constraints из профиля автоматически
3. **Изоляция TaskContext** — рабочая память привязана к сессии, профиль — глобальный
