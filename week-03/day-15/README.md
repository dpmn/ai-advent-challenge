# Неделя 3 — День 15. Контролируемые переходы состояний

## Задание

Реализуйте **явные переходы между состояниями задачи**.

**Сделайте так, чтобы:**
- у задачи были допустимые состояния
- были разрешённые переходы между ними
- ассистент не мог "перепрыгнуть" этап

**Пример:**
- нельзя делать реализацию до утверждённого плана
- нельзя делать финал без валидации

**Проверьте:**
- попытки перейти в недопустимое состояние
- реакцию ассистента
- корректность продолжения после паузы

---

## Реализация

State Machine была реализована ранее в `agents/state_machine.py`. Для задания дня 15 выполнена проверка соответствия требованиям — все три пункта полностью покрыты существующим кодом.

**Ключевые файлы:**
- `agents/state_machine.py` — `AgentState`, `ALLOWED_TRANSITIONS`, `PipelineAgent.transition_to()`, `StageAgent`
- `agents/jarvis.py` — интеграция SM в основной агент, сохранение состояния в SQLite

**Что реализовано:**
- 4 состояния: `PLANNING → EXECUTION → VALIDATION → DONE`
- Карта разрешённых переходов `ALLOWED_TRANSITIONS` — нелегальные переходы блокируются в коде
- `transition_to()` возвращает ошибку при попытке перепрыгнуть этап
- Каждый `StageAgent` имеет изолированную историю, артефакты передаются между этапами
- Состояние сохраняется в SQLite — агент продолжает с того же этапа после перезапуска

## Сценарий проверки

### Подготовка

1. Запустить сервер: `python ./webui/app.py`
2. Открыть браузер: http://127.0.0.1:5000
3. Создать новую SM-сессию — написать в чат:
   ```
   /new sm День 15
   ```
   Убедиться, что ответ содержит `[SM]` и название сессии.

4. Проверить начальный статус:
   ```
   /sm
   ```
   Ожидаемый ответ: этап `PLANNING`, валидация `вкл`.

---

### Шаг 1. Попытка перепрыгнуть этап (недопустимый переход)

Попробовать перейти сразу в `DONE`, минуя все промежуточные этапы:
```
/step DONE
```
**Ожидаемый результат:** сообщение об ошибке — переход запрещён, указаны допустимые состояния (`EXECUTION`).

Попробовать перейти в `VALIDATION`:
```
/step VALIDATION
```
**Ожидаемый результат:** аналогичная ошибка — из `PLANNING` нельзя перейти в `VALIDATION`.

---

### Шаг 2. Работа в этапе PLANNING

Отправить задачу агенту (ассистент работает только в рамках текущего этапа — планирует, не реализует):
```
Нужно написать Python-скрипт, который считает факториал числа
```
**Ожидаемый результат:** ответ агента в режиме `[PLANNING]` — план реализации (шаги, структура), без готового кода.

Проверить статус:
```
/sm
```

---

### Шаг 3. Переход PLANNING → EXECUTION

Перейти на следующий этап:
```
/step EXECUTION
```
**Ожидаемый результат:** сообщение "Переход на этап EXECUTION выполнен."

Теперь попробовать недопустимый переход назад через один:
```
/step DONE
```
**Ожидаемый результат:** ошибка — из `EXECUTION` нельзя перейти в `DONE`, допустимые: `VALIDATION`, `PLANNING`.

---

### Шаг 4. Работа в этапе EXECUTION

Отправить запрос на реализацию (агент получит артефакт плана из предыдущего этапа):
```
Реализуй план
```
**Ожидаемый результат:** ответ агента в режиме `[EXECUTION]` — готовый код с учётом плана из PLANNING.

---

### Шаг 5. Переход EXECUTION → VALIDATION

```
/step VALIDATION
```
**Ожидаемый результат:** сообщение об успешном переходе.

Запустить валидацию:
```
Проверь результат на соответствие плану
```
**Ожидаемый результат:** ответ агента в режиме `[VALIDATION]` — анализ кода, замечания или подтверждение соответствия.

---

### Шаг 6. Попытка перейти в PLANNING из VALIDATION (возврат)

```
/step PLANNING
```
**Ожидаемый результат:** ошибка — из `VALIDATION` нельзя перейти в `PLANNING`. Допустимые: `DONE`, `EXECUTION`.

---

### Шаг 7. Завершение и проверка корректности после паузы

Перейти в `DONE`:
```
/step DONE
```

Запросить итог:
```
Подведи итог
```
**Ожидаемый результат:** ответ в режиме `[DONE]` — краткое резюме всех этапов.

**Проверка корректности после паузы:**
- Закрыть браузер, перезапустить сервер: `python ./webui/app.py`
- Открыть http://127.0.0.1:5000 — сессия должна автоматически восстановиться
- Написать:
  ```
  /sm
  ```
  **Ожидаемый результат:** этап `DONE`, артефакты всех этапов сохранены. Агент продолжает с того же состояния, где остановился.
