# Неделя 3 — День 13. Конечный автомат (State Machine)

## Задание

**Реализуй состояние задачи как конечный автомат:**
- этап задачи
- текущий шаг
- ожидаемое действие

**Особенности:**
- режим State Machine (вкл / выкл) определяется при создании сессии, после не меняется
- должна быть галочка валидировать / не валидировать результат перед переходом на следующий шаг пайплайна
- если запущен режим без валидации, то состояния должны переключаться сами (задача должна самостоятельно пройти по всем этапам: родился артефакт -> переходит в следующую часть)
- берём базовый набор состояний пайплайна: planning → execution → validation → done
- для каждого стостояния должен работать отдельный StageAgent
  - Изолированная история (только сообщения своего этапа) + свой system prompt
  - Может использовать другую модель (например, для планирования — Qwen3-30B-A3B, для кода — Qwen3-Coder-Next) - по умолчанию ставить базовую дешёвую модель
  - Параметры LLM (temperature, max_tokens): возможность задавать отдельные для каждого этапа, по умолчанию - общие для сессии
  - При ре-входе на этап (через /step PLANNING) восстанавливает свою историю
- для оркестрации должен быть PipelineAgent
  - Создаётся при инициализации сессии с SM
  - Содержит 4 StageAgent (по одному на этап)
  - Принимает user_input, маршрутизирует текущему StageAgent'у, управляет переходами
  - Хранит артефакты этапов (plan, code, validation_result) и передаёт их между агентами

**Примечание:** _возможно стоит сделать реализацию классов через несколько модулей, чтобы потом было легче поддерживать.
Пиши доксринг для классов и методов._

**Проверить:**
- паузу на любом этапе
- продолжение без повторных объяснений

## Сценарий для видео

Запись через WebUI (http://127.0.0.1:5000). Демонстрируем State Machine: создание SM-сессии, ручной режим с валидацией, авто-прогрессию, ре-вход на этап.

### Подготовка к записи

1. Убедись, что `.env` содержит `CLOUDRU_SECRET_KEY`.
2. Запусти Flask: `python webui/app.py`
3. Открой в браузере: http://127.0.0.1:5000
4. Удали старые сессии через ✕.
   **Начать запись видео.**

## Ожидаемые критерии

1. **State Machine создаётся при создании сессии** — `/new sm [name]` или `name sm` в UI
2. **4 этапа: PLANNING → EXECUTION → VALIDATION → DONE** с изолированными StageAgent'ами
3. **Ручной режим (с валидацией)** — `/step STAGE` для перехода, пауза на любом этапе
4. **Авто-режим (без валидации)** — `/sm validate off` → один запрос проходит все 4 этапа
5. **Запрещённые переходы** — блокируются `ALLOWED_TRANSITIONS`
6. **Ре-вход на этап** — `/step PLANNING` восстанавливает историю этапа
7. **SM-режим неизменяем** — после создания сессии нельзя включить/выключить SM
8. **Артефакты этапов** — `/artifact` показывает результаты всех пройденных этапов
9. **Разные модели на этапах** — PLANNING: Qwen3-30B-A3B, EXECUTION: Qwen3-Coder-Next (по умолчанию)
